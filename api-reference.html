    Plejady

    Plejady keeps the contexts that define your domain and business logic.

    Plejady.Accounts

    Provides logic for managing accounts. This includes users, guests, admins, etc.

    Plejady.Accounts.Guest

    Provides logic for guest users (users whose email is not a student's email).

    Plejady.Accounts.SuggestedAdmin

    Provides logic for suggested admins (users that will automatically become admins).

    Plejady.Accounts.Token

    Utilities for managing user tokens.

    Plejady.Accounts.User

    Provides logic for managing users.

    Plejady.Accounts.User.Promotion

    Provides logic for promoting users.

    Plejady.Accounts.UserToken

    Provides logic for managing user tokens.

    Plejady.CacheInitiator

    This module is responsible for initializing the cache with the data from the database.

    Plejady.Cldr

    Provides the core functions to retrieve and manage the CLDR data that supports formatting and localisation.

    Plejady.Cldr.AcceptLanguage

    Parses HTTP Accept-Language header values as defined in rfc2616.

    Plejady.Cldr.Calendar

    Data functions to retrieve localised calendar information.

    Plejady.Cldr.Currency
    Plejady.Cldr.Date
    Plejady.Cldr.Date.Interval

    Interval formats allow for software to format intervals like "Jan 10-12, 2008" as a shorter and more natural format than "Jan 10, 2008 - Jan 12, 2008". They are designed to take a start and end date, time or datetime plus a formatting pattern and use that information to produce a localized format.

    Plejady.Cldr.DateTime.Format

    Plejady.Cldr.DateTime.Formatter

    Implements the compilation and execution of date, time and datetime formats.

    Plejady.Cldr.Locale

    Backend module that provides functions to define new locales and display human-readable locale names for presentation purposes.

    Plejady.Cldr.Number.Ordinal

    Plejady.Config

    Provides the model and logic for the application config.

    Plejady.Config.Schema

    Model and helpers for the Plejady.Config schema.

    Plejady.DataCase

    This module defines the setup for tests requiring access to the application's data layer.

    Plejady.Presentation

    Provides the model and logic for presentations.

    Plejady.Registration

    Provides the model and logic for registrations.

    Plejady.Registry

    This module is responsible for managing the registry of presentations and their occupancy. It provides a super fast read-write cache for the real-time component of the app with fire-and-forget requests to the database for permanent storage.

    Plejady.Release

    Used for executing DB release tasks when run in production without Mix installed.

    Plejady.Repo

    The Ecto repository for this application.

    Plejady.Room

    Provides the model and logic for rooms.

    Plejady.TestingFixtures

    This module defines test helpers for creating entities via the Plejady.Testing context.

    Plejady.Timeblock

    Provides the model and logic for timeblocks.

    Plejady.TimedRelease

    GenServer for handling timed release of the application.

    PlejadyWeb

    The entrypoint for defining your web interface, such as controllers, components, channels, and so on.

    PlejadyWeb.AuthController

    Controller for handling authentication. Uses Ueberauth to simplify the code in the app itself.

    PlejadyWeb.ConnCase

    This module defines the test case to be used by tests that require setting up a connection.

    PlejadyWeb.CoreComponents

    Provides core UI components.

    PlejadyWeb.Endpoint

    The endpoint for the PlejadyWeb application. Encapsulates all socket and plug configuration.

    PlejadyWeb.Header

    Component library for the app header.

    PlejadyWeb.UserAuth

    Provides LiveView hooks and helper functions for user authentication.
